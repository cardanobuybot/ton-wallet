# ton-wallet — правила проекта

Некастодиальный веб-кошелёк для TON (PWA). Позиционирование: «кошелёк, который не даёт
обмануть» — анти-скам проверки и симуляция транзакций перед подписью.

## Стек (зафиксирован)

- Монорепо на npm workspaces: `packages/core`, `apps/web`, `apps/api`
- `packages/core`: чистый TypeScript; runtime-зависимости только `@ton/ton`, `@ton/core`,
  `@ton/crypto`, `tweetnacl` (одобрен для TON Connect); тесты — vitest
- `apps/web`: Vite + React + TS, PWA (manifest + service worker), хостинг Vercel
- `apps/api`: Fastify + TS, хостинг Railway
- TypeScript strict везде, ESLint + Prettier, Node >= 22

## Правила

1. **Ключи только на клиенте.** Приватные ключи и мнемоники существуют только в клиентском
   коде (`packages/core`, `apps/web`). В `apps/api` — никогда: ни ключей, ни подписи.
   Ключевой материал не логируется и не попадает в тесты снапшотами.
2. **Криптография — только с тестами.** Любой код, касающийся криптографии, сериализации
   или адресов, добавляется только вместе с тестами.
3. **Лицензия клиента — GPL-3.0.** Разрешено адаптировать код из
   mytonwallet-org/mytonwallet (GPL-3.0) с указанием источника в комментарии.
   Из несовместимых по лицензии источников не копировать.
4. **Новые зависимости — только по согласованию с владельцем.** Lockfile коммитится.
5. **Только testnet**, пока владелец явно не разрешит mainnet.
6. **Новые кошельки — только W5** (`WalletContractV5R1`). Импорт v4R2/v3R2 появится позже,
   поэтому ядро спроектировано под несколько версий контрактов (registry версий).

## Service worker (apps/web)

Максимально консервативный: HTML и навигация — только network-first (`index.html` никогда
не отдаётся из кэша при живой сети); кэшировать можно только хэшированные статические
ассеты. Мгновенная доставка обновлений — свойство безопасности кошелька.

## Команды

- Установка: `npm install` (в корне)
- Typecheck: `npm run typecheck`
- Lint: `npm run lint`
- Тесты: `npm test`
